Casus

De YuGiOh card collector casus gaat over een website waar je een account kan aanmaken om vervolgens YuGiOh kaarten te gaan verzamelen!
De hoofdfunctionaliteit van de website is het aanmaken van collecties, waarmee je bij kan houden wat voor kaartje je alemaal hebt.
Verder is het ook mogelijk om met de verzamelde kaarten een deck te maken en die bij te houden op de website.
Naast dit alles is er ook nog een ruil functionaliteit. Hiermee kan je met andere gebruikers ruilen om zo je collectie vol te krijgen!

Entiteiten

Binnen de casus worden er een aantal entiteiten gebruikt om functionaliteit en vorm te geven aan de card collector. website

De volgende entiteiten worden gebruik in deze casus:

Users

Users zijn nodig om de website te laten werken. Een user kan een account aan maken om vervolgens de functionaliteiten die de website te bieden heeft te gebruiken.

Een user bestaat uit de volgende attributen:

  • Users
    • Id
    • Firstname
    • Lastname
    • Username
    • Email

    Alle attributen zijn hier verplicht voor het aanmaken van een user.

Cards

Om een collectie website te kunnen maken zijn er kaarten nodig, deze kaarten zijn statisch ingeladen data vanuit de ygoprodeck api

Cards bestaan uit de volgende attributen:

  • Cards
    • Id
    • Name
    • Type
    • Description
    • Archetype
    • Race
    • Image (Niet een image zelf maar de link naar de image toe die via Ygoprodeck's https://storage.googleapis.com link gaat anders resulteerd het in een ip blacklist
    • CardSetName (De naam van de set waar de kaart uit komt.)
    • CardSetCode (De code die de kaart heeft binnen de set. e.g. BODE-EN030)
    • Attack
    • Defense
    • Level
    • Attribute
    • obtained (Dit attribuut wordt gebruikt om later in een collectie aan te kunnen duiden of je de kaart hebt.)

Sets

cards behoren tot een uitgebrachte set. Deze sets zijn ook statisch ingeladen data vanuit de ygoprodeck api

Sets bestaan uit de volgende attributen:

  • Sets
    • Id
    • SetName (De naam van de set.)
    • SetCode (De afkorting van de set die op de kaarten gebruikt wordt E.g. Burst Of Destiny -> BODE)
    • NumberOfCards
    • ReleaseDate

Collections

Een collection kan vergeleken worden met een set, Enkel bestaat een set uit wat data over de set zelf en niet over de inhoudelijke kaarten. Een collectie kan je aanmaken, de kaarten die erin zitten aanklikken om te laten zien dat je ze hebt en je kan de kaarten gebruiken om ze in je deck te stoppen. Op een collectie zullen de volledige CRUD functionaliteiten gebruikt worden.

Een collection bestaat uit de volgende attributen:

  • Collections
    • Id
    • CollectionName
    • CollectionSetName (Dit is een string die overheen zal komen met de namen van de beschikbare sets zodat de kaarten hiervan opgehaald kunnen worden.)
    • CreationDate
    • NumberOfCards
    • Cards (De array van kaarten die in de geselecteerde set zit. Per kaart zelf is te zien of je de kaart al hebt.)

Decks

Een deck is een principe binnen YuGiOh waarin je kaarten samensteld naar jou keuze om vervolgens een match tegen iemand te spelen met een ander deck Een deck bestaat uit een main deck, extra deck en een side deck.

Het deck entiteit heeft volgende attributen:

  • Decks
    • Id
    • DeckName
    • NumberOfCards
    • MainDeck (Dit is een array van cards die het main deck vormen.)
    • ExtraDeck (Dit is een array van cards die het extra deck vormen.)
    • sideDeck (Dit is een array van cards die het side deck vormen.)
    • CreationDate

Trades

Het is mogelijk om op de card collector website te traden met andere gebruikers. je kan zowel losse kaarten als volledige decks ruilen met elkaar.

Een trade bestaat uit de volgende attributen:

  • Trades
    • Id
    • Trader (De user die de trade aanmaakt.)
    • Tradee (De user waarmee de geruild wil worden.)
    • CardToTrade (De kaart die de user wilt ruilen.)
    • CardToReceive (De kaart die de user wilt hebben van iemand anders.))
    • DeckToTrade (Het deck dat de user wilt ruilen.)
    • DeckToReceive (Het deck dat de user van iemand anders wilt hebben.)
    • CreationDate
